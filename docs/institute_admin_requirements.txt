2.5 Security Architecture  
• HTTPS enforced on all endpoints (TLS 1.3 minimum); HTTP auto -redirects to HTTPS via Cloudflare  
• JWT Auth: 8 -hour access token + 30 -day refresh token with rotation on each use  
• Passwords: bcrypt hashed, cost factor 12 minimum — never stored in plaintext  
• SQL Injection: Laravel Eloquent ORM with parameterized queries exclusively — no raw SQL  
• XSS Prevention: all user inputs sanitized server -side; Content Security Policy headers on all pages  
• File uploads: MIME type server -side verification + ClamAV virus scan before storage  
• Rate limiting: max 5 failed login attempts per IP per 15 minutes, then CAPTCHA challenge  
• 2FA: OTP SMS via Sparrow SMS for Super Admin and Institute Admin accounts  
• Tenant isolation: tenant_id enforced at middleware level — every DB query scoped to current tenant 
via Laravel global scope  
• Admin audit log: all create/update/delete actions logged with user_id, IP, timestamp, and JSON diff 
of changed data  
• Student PII (citizenship number, address) encrypted at rest using AES -256 
• Document files stored in private S3/Wasabi bucket — signed URLs with 15 -minute expiry  
• Penetration testing: scheduled every 6 months pre -launch and annually post -launch

Hamro Labs Academic ERP  |  Page 25 of 34 SECTION 7 — Security & Compliance  
7.1 Authentication & Authorization  
• JWT: 8 -hour access token + 30 -day refresh token. Refresh tokens rotated on every use. Token 
invalidated immediately on logout.  
• RBAC enforced at middleware layer: role extracted from JWT payload, permission matrix checked 
before controller execution  
• 2FA via OTP SMS (Sparrow SMS) mandatory for Super Admin and Institute Admin accounts  
• Rate limiting: 5 failed login attempts per IP per 15 minutes → CAPTCHA challenge required  
• Session management: stateless design (sessions in Redis, not server memory) — supports 
horizontal scaling  
 
7.2 Data Isolation  
• Laravel Global Scope applied to all Eloquent models: every DB query automatically includes WHERE 
tenant_id = ? clause  
• Middleware validates JWT tenant_id matches requested resource tenant — no cross -tenant access 
possible  
• Enterprise plan option: dedicated MySQL instance for complete physical data isolation  
• No shared application state between tenants: Redis keys prefixed with tenant_id  
 
7.3 Encryption  
• All data in transit: HTTPS with TLS 1.3 minimum. Cloudflare enforces HTTPS at edge.  
• Student PII at rest: AES -256 encryption for citizenship number, national ID, address fields  
• Document storage: private S3/Wasabi bucket with server -side AES -256 encryption. Signed URLs 
with 15 -minute expiry.  
• Passwords: bcrypt hashed, cost factor 12 minimum — never stored, never logged  
 
7.4 Activity Audit Logs  
• All Admin create/update/delete actions logged: audit_logs table with user_id, IP, timestamp, table, 
record_id, JSON diff of old vs. new values  
• Super Admin impersonation sessions fully logged with start/end time, actions taken, and reason  
• SMS logs: full delivery status trail per message (gateway, status, timestamp, recipient)  
• File access logs: every document download with signed URL tracked by student/staff user  
• Logs are immutable — no user (including Super Admin) can delete audit records  
 
7.5 Backup Strategy  
• MySQL: automated backup every 6 hours via DigitalOcean Managed Database (or mysqldump to 
encrypted S3 on self -managed)  
• Retention: 7 -day rolling daily backups + 4 -week rolling weekly snapshots + 12 -month monthly 
archives  
• File storage: S3/Wasabi versioning enabled with 30 -day version retention  
• Recovery targets: RTO 4 hours / RPO 6 hours

9.1 Authentication and Authorization  
• JWT: 8 -hour access token + 30 -day refresh token. Refresh tokens rotated on every use. Token 
invalidated immediately on logout.

9.2 Data Isolation  
• Laravel Global Scope applied to all Eloquent models: every DB query automatically includes 
WHERE tenant_id = ? clause  
• Middleware validates JWT tenant_id matches requested resource tenant — no cross -tenant 
access possible  
• Enterprise plan option: dedicated MySQL instance for complete physical data isolation  
• No shared application state between tenants: Redis keys prefixed with tenant_id  
 
9.3 Encryption  
• All data in transit: HTTPS with TLS 1.3 minimum. Cloudflare enforces HTTPS at edge.  
• Student PII at rest: AES -256 encryption for citizenship number, national ID, and address fields  
• Document storage: private S3/Wasabi bucket with server -side AES -256 encryption. Signed 
URLs with 15 -minute expiry.  
• Passwords: bcrypt hashed, cost factor 12 minimum — never stored, never logged  
 
9.4 Audit Logging  
• All Admin create/update/delete actions logged: audit_logs table with user_id, IP, timestamp, 
table name, record_id, JSON diff of old vs. new values  
• Super Admin impersonation sessions fully logged with start/end time, actions taken, and reason  
• SMS logs: full delivery status trail per message (gateway, status, timestamp, recipient number)  
• File access logs: every document download with signed URL tracked by student/staff user  
• Audit logs are immutable — no user including Super Admin can delete audit records  
 
9.5 Input Validation and Attack Prevention  
• SQL injection: Laravel Eloquent ORM with parameterized queries exclusively. Raw SQL queries 
strictly prohibited in codebase.
